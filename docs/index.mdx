**컴포넌트 시대**를 위한\
마크다운
=================

MDX를 사용하면 마크다운 콘텐츠에 JSX를 사용할 수 있습니다.
인터랙티브 차트나 알림과 같은 컴포넌트를 가져와 콘텐츠 내에 임베드할 수 있습니다.
이를 통해 컴포넌트와 함께 긴 형식의 콘텐츠를 작성하는 것이 즐거워집니다. {/* more */}
🚀

## MDX는 무엇을 하나요?

<div className="big-columns">
  <div>
    JSX를 통해 임베디드 컴포넌트와 함께 마크다운을 작성합니다:

    ```mdx path="example.mdx"
    import {Chart} from './snowfall.js'
    export const year = 2023

    # Last year's snowfall

    In {year}, the snowfall was above average.
    It was followed by a warm spring which caused
    flood conditions in many of the nearby rivers.

    <Chart year={year} color="#fcb32c" />
    ```
  </div>

## 시작하기

대부분의 번들러, 프레임워크, 에디터에 대한 통합이 있습니다.
Docusaurus, Next.js, 또는 Vite로 빌드하든,
Rollup, esbuild, 또는 webpack을 선호하든,
React, Preact, 또는 Vue를 사용하든 상관없습니다.

## MDX 요약

<div className="emoji-list">
  * ❤️ **강력함**: MDX는 마크다운과 JSX 구문을 혼합하여 JSX 기반 프로젝트에 완벽하게 맞춥니다
  * 💻 **모든 것이 컴포넌트**: MDX에서 기존 컴포넌트를 사용하고 다른 MDX 파일을 컴포넌트로 가져옵니다
  * 🔧 **커스터마이징 가능**: 각 마크다운 구조에 대해 렌더링될 컴포넌트를 결정합니다 (`{h1: MyHeading}`)
  * 📚 **마크다운 기반**: 마크다운의 단순함과 우아함은 그대로 유지되며, 원할 때만 JSX를 사용합니다
  * 🔥 **엄청나게 빠름**: MDX는 런타임이 없으며, 모든 컴파일은 빌드 단계에서 이루어집니다
</div>
