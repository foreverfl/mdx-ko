# 컴포넌트

MDX의 훌륭한 점은 마크다운을 작성하고 HTML 요소 대신 사용할 컴포넌트를 지정할 수 있다는 것입니다. {/* more */}
다음 표는 덮어쓸 수 있는 HTML 요소, 필요한 마크다운, 그리고 일반적으로 대응되는 JSX 컴포넌트를 나열합니다.
하지만 첫 번째 행의 `a`를 예로 들면, 다음과 같이 하여 해당 하이퍼링크를 `<FancyLink>`로 대체할 수 있습니다:

```tsx
import Readme from './readme.md' // Assumes an integration is used to compile MDX -> JS.
import {FancyLink} from './components/fancy-link.js'

<Readme components={{a: FancyLink}} />
```

덮어쓸 수 있는 컴포넌트는 표준 HTML 이름을 사용합니다.
일반적으로 마크다운에서 이러한 이름들은: `a`, `blockquote`, `br`, `code`, `em`,
`h1`, `h2`, `h3`, `h4`, `h5`, `h6`, `hr`, `img`, `li`, `ol`, `p`, `pre`,
`strong`, 그리고 `ul`입니다.
[`remark-gfm`][gfm]을 사용하면 ([가이드 참조][guide-gfm]), 다음도 사용할 수 있습니다: `del`,
`input`, `section`, `sup`, `table`, `tbody`, `td`, `th`, `thead`, 그리고 `tr`.
rehype와 함께 작동한다고 광고하는 새로운 구조를 지원하는 다른 remark 플러그인들도 MDX와 함께 작동할 것입니다.

컴포넌트에 대한 자세한 정보는
[¶ MDX 사용하기의 컴포넌트 섹션][components]에서 확인할 수 있습니다.
